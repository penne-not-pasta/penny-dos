# To Contribute...

To contribute to Penny-DOS please do (*or be*) the following

- 13 (or older)
- Have a GitHub account
- Fork the repository
- Create a new branch with a name following the guide in the `/Penny-Dos/docs-for-devs/guide.md` file.
- Create a pull request with a descriptive title and a detailed description of the changes you made.
- Ensure your code follows the style guide and passes all tests.
- Ensure your code is well-documented and follows best practices.
- Ensure your code is tested and passes all tests.
- Ensure your code is optimized for performance.
- Ensure your code is secure and follows best practices.
- Ensure your code is maintainable and follows best practices.
- Ensure your code is scalable and follows best practices.
- Ensure your code is extensible and follows best practices.
- Ensure your code is reusable and follows best practices.
- Ensure your code is modular and follows best practices.
- Ensure your code is safe and follows best practices.
- Ensure your code is efficient and follows best practices.
