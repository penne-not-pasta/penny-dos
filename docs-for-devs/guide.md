# Guide.

## 0. Branch Naming Conventions.

Example: `[YOUR-NAME] - Feature/Change, vX.X.X-x`

*Your branch must have your name, feature/change name, version number.*

## 1.PR Conventions.

Your pull request must follow the naming conventions above in section `0`.

### 1.1. PR Content Conventions.

The contents of your full request must be formatted as follows:

```markdown
  # v.X.X.X-x - Feature/Change
  
  *changes made by <NAME>, on <DATE>*
  
  ## Change List
  
  - change 1
  - change 2
  - change 3
  - change 4
  - change 5, 6, 7, so on so forth...
  
  ## Files I Edited:
  
  - /penny-dos/folder/file1.ext
  
  ## Files I Created:
  
  - /penny-dos/folder/file2.ext
```
